What is UNIX?

UNIX is a family of operating systems. An operating system is a set of programs that act as a link between the computer and the user.

Of all the computer programs, the subset that allocates the system resources and coordinates all the details of the computer's internals,
is called the kernel.

Users communicate with the kernel through a program known as the shell. The shell is a command line interpreter; it translates commands entered by the user and converts them into a language that is understood by the kernel.

UNIX Architecture Elements

  • Kernel: The kernel is the heart of the operating system. It interacts with hardware and most of the tasks like memory management, task scheduling and file management.
  • Shell: The shell is the utility that processes your requests. When you type in a command at your terminal, the shell interprets the command and calls the program that you want. The shell uses standard syntax for all commands. C Shell, Bourne Again Shell and Korn Shell are most famous shells which are available with most of the Unix variants.
  • Commands and Utilities: There are various commands and utilities which you would use in your day to day activities. cp, mv, cat and grep etc. are few examples of commands and utilities. There are over 250 standard commands plus numerous others provided through 3rd party software. All the commands come along with various optional options.
  • Files and Directories: All data in UNIX is organized into files. All files are organized into directories. These directories are organized into a tree-like structure called the filesystem.

UNIX FileSystem

File management 101

All data in UNIX is organized into files. All files are organized into directories. These directories are organized into a hierarchical tree-like structure called the filesystem.

A directory is a file whose sole job is to store file names and related information. All files, whether ordinary, special, or directories, are contained within a directory.

Root Directory

Directories are arranged in a hierarchy with root (/) at the top.

Navigate to the root folder of the bootcamp server using cd: 


$ cd /

Check where you are within the filesystem hierarchy at any time with pwd: 


$ pwd
/

Try listing the files within the root folder with ls: 


$ ls
bin   dev  home        lib    lost+found  mnt  proc  run   srv  tmp  var
boot  etc  initrd.img  lib64  media       opt  root  sbin  sys  usr  vmlinuz

Home Directory

The directory in which you find yourself when you first login is called your home directory.

You will be doing much of your work in your home directory and subdirectories that you'll be creating to organize your files.

Navigate back to your home folder: 


$ cd ~

Absolute/Relative pathnames

Again, directories are arranged in a hierarchy tree with root (/) at the top. The position of any file within the hierarchy is described by its pathname.

Elements of a pathname are separated by a /. A pathname is absolute if it is described in relation to root, so absolute pathnames always begin with a /.

Here are some examples of absolute pathnames: 


/etc/apache2/apache2.conf
/var/www/warmup/

A pathname can also be relative to your current working directory. Relative pathnames never begin with /. Relative to campelo's home directory, some pathnames might look like this: 


Development/warmups
codecadets/resources/unix-tutorial.txt

Creating Directories

In the bootcamp server, within your home directory, create a new folder named warmup: 


$ mkdir warmup

The mkdir command produces no output if it successfully creates the requested directory.

Sometimes when you want to create a directory, its parent directory or directories might not exist. In this case, mkdir issues an error message as follows: 


$ mkdir temp/resources/
mkdir: cannot create directory 'temp/resources/': No such file or directory

In such cases, you can specify the -p option to the mkdir command. It creates all the necessary directories for you: 


$ mkdir -p temp/resources/

Removing Directories

Use rmdir to remove the empty directory temp/resources: 


$ rmdir temp/resources

With rmdir (and mkdir), you can remove (create) multiple directories at the same time as follows: 


$ mkdir development code-breaks temp-folder
$ rmdir code-breaks

Moving/Renaming Files

The mv (move) command can be used both to move or rename files (and directories in particular).

You will be renaming/moving a file depending on if you use mv to the same/different parent folder.

Rename the temp directory to tmp:

Metacharacters

Metacharacters are a group of characters that have special meanings to the UNIX operating system. For instance, * represents 0 or more characters, and ? represents (any) single character.

Symbol Meaning
* Matches any number of any characters
? Matches any single character
\ char Ignore shell's special meaning of char
command < file Input the content of file to the command
command > file Send the output of command to the end of the file
command_1 | command_2 Sends the output of command_1 as the input of command_2

The . & .. Directories

The . (dot) filename represents the current working directory;
and the .. (dot dot) filename represents the directory one level above the current working directory, often referred to as the parent directory.

Every directory on the UNIX filesystem contains both the . and the .. filenames.

Every file (or folder) that has a name starting with . is hidden.

To list all files (including hidden ones) inside the tmp directory you just created use the -a(all) option of ls: 


$ ls -a tmp/
.  ..

UNIX Environment

An important UNIX concept is the environment, which is defined by environment variables. Some environment variables are set by the system, others by you, yet others by the shell, or any program that loads another program.

A variable is a character string to which we assign a value. The value assigned could be a number, text, filename, device, or any other type of data.

Variables

Set your first UNIX environment variable and print its value using echo: 


$ TEST="UNIX tutorial"
$ echo $TEST
UNIX tutorial

Note that environment variables are set without using $ sign but while accessing them we use $ sign as a prefix. These variables retain their values until we come out of the shell.

When you login to the system, the shell undergoes a phase called initialization to set up the environment. This is usually a two step process that involves the shell reading the following files:

  • /etc/profile
  • ~/.profile

Shell Initialization

These are the steps involved in the shell initialization:

  • The shell checks to see whether the file /etc/profile exists;
  • If it exists, the shell reads it. Otherwise, this file is skipped and no error message is displayed;
  • The shell checks to see whether the file .profile exists in your home directory;
  • If it exists, the shell reads it; otherwise, the shell skips it and no error message is displayed.

As soon as these steps are done, the shell displays the command line prompt: 


$

Profile Files

The file /etc/profile is maintained by the system administrator of your UNIX machine and contains shell initialization information required by all users on a system.

The file .profile is under your control. You can add as much shell customization information as you want to this file. The minimum set of information that you need to configure includes:

  • The type of terminal you are using;
  • A list of directories in which to locate commands;
  • A list of variables affecting the look and feel of your terminal.

Check your .profile using vim: 


$ vim ~/.profile

PATH Variable

The PATH variable specifies the location in which the shell looks for commands.

Try printing your PATH variable in the console: 


$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/local/games:/usr/games

PS1 Variable

The characters that the shell displays as your command prompt are stored in a environment variable called PS1. You can change this variable to be anything you want. As soon as you change it, it'll be used in the current shell session from that point on.

Try changing the PS1 to the following: 


$ PS1="\u@\h \w\$ "
campelo@guincho ~$

If all went well, the command prompt of your current shell session has become something like campelo@guincho ~$.

Now make that change load automatically when the shell initializes by adding the PS1 variable to ~/.profile. 


campelo@guincho ~$ vim ~/.profile

Filesystem security (access rights)

Access rights is an important component of UNIX that provides a secure method for storing files. Every file in UNIX has the following attributes:

  • Owner permissions: The owner's permissions determine what actions the owner of the file can perform on the file;
  • Group permissions : The group's permissions determine what actions a user, who is a member of the group that a file belongs to, can perform on the file;
  • Other (world) permissions: The permissions for others indicate what action all other users can perform on the file.

File Permissions

Try long listing the unix cheat-sheet.txt file you copied before: 


campelo@guincho:~$ cd warmup/unix/
campelo@guincho:~/warmup/unix$ ls -l cheat-sheet.txt
-rw-r--r-- 1 campelo campelo 8638 Jan 11 15:11 cheat-sheet.txt

Here the first column (with the exception of the first character) represents different access modes (permissions) associated with the file.

The permissions are broken into groups of threes. Each position in a group denotes a specific permission. In this order, read (r), write(w) and execute(x).
