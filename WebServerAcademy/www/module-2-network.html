Computer Networks

  • Used to Connect Machines to other Machines
  • Can be categorized considering distance and characteristics

Wide Area Network

  • Communication over large distances, spanning entire continents and joining computers across an ocean;
  • Operates at slower speeds and exhibits significant delay;
  • Topology consists of a mesh of individual point-to-point links.

Internet Backbone in Europe
  • MEO Router uplink labelled as WAN. Not correct, but it is a connection to the outside of the LAN..
  • Delay - time it takes for a bit to travel from a source to it's destination
  • Latency is a broad engineering concept, can mean the same as delay or not, depends on context - time between stimulus and it's response, always smaller or equal than the speed of the light
  • Latency in a switch is the delay between the time when a device receives a frame and the time that frame is forwarded out the destination port
  • RTT - Round Trip Time

Local Area Network

  • Short distances only;
  • Operates at high speeds and does not exhibit significant delay;
  • Each computer contains a network interface that connects the machine to the network medium.
network mediums can be copper, coaxial or radio

Ethernet

Invented at Xerox in the 1970s, and has become the most popular means for connecting computers in a LAN. Evolved from coaxial cable to twisted pair and fiber optics. Transmission speeds range from the original 10 Mbit/s to current 100Mbit/s, 1Gbit/s and 10Gbit/s.

Bus Topology

  • All computers share a single communication channel;
    •
  • Broadcast-based, all computers receive all transmissions;
  • Each computer discards transmissions destined to other computers;
  • Collisions can occur and need to be handled;
  • 10 Base 5 for 10 Mbit/s half-duplex on thick coaxial cable;
  • 10 Base 2 for 10 Mbit/s half-duplex on thin coaxial cable.

Ethernet Hub

  • Replaces the coaxial bus by acting as a single network segment;
  • Connects multiple Ethernet devices together;
  • 10 Base-T for 10 Mbit/s half-duplex on unshielded twisted pair cabling.

Ethernet Switch

  • Replaces shared medium with dedicated full duplex links;
  • Prevents collisions;
  • 100 Base-TX for 100 Mbit/s full-duplex on shielded twisted pair cabling.

Ethernet Address

A Media Access Control (MAC) address is used as a global unique identifier assigned to ethernet network interfaces for communications on the physical network segment

  • 48-bit addressing scheme
  • MAC address encodes the manufacturer identification number

The Internet

Internet Architecture

We have seen how machines connect to individual networks...
But how are networks interconnected to form an internetwork?

  • We need computers that connect to multiple networks;
  • We need these computers to forward packets between networks;
  • Such computers are called Routers;
  • Forwarding network traffic among networks is called Routing.

Internet Protocol

  • Abstracts underlying LAN technology, making the internet appear to be a single uniform entity;
  • Has the task of delivering packets from a source host to a destination host;
  • IPv4 is the dominant protocol of the Internet;
  • IPv6 is its successor.
IPv4 addresses were exhausting, IPv6 allowed for 100k address per mˆ2 of earth surface, but a very successful patch was applied to v4 (Classless/NAT/PAT), which caused v6 adoption to slow down.

Internet Addresses

Each host on the internet is assigned a unique 32-bit internet address that is used in all communication with that host

IP addresses encode both a network and a host on that network. They do not specify an individual computer, but a connection to a network

If a host computer moves from one network to another, its IP address must change

  1. A host is a network node that may expose information resources, services or applications;
  2. Every host is a physical network node, but not every physical network node is a host;
  3. Some network devices as modems, hubs and switches are NOT considered hosts;

Subnet Masks

The parts of the IP address that are used as the network and host addresses are not fixed, so the network and host addresses cannot be determined unless you have more information.

This information is supplied in another 32-bit number called a subnet mask.

Network and Broadcast

The first address of a subnetwork is reserved for addressing the network itself

The last address of a subnetwork is reserved for broadcasting to all hosts on the network

Loopback Address

All addresses starting with 127 are reserved for testing and inter-process communication on the local machine. They are called loopback addresses and should never appear on the network

When a software program uses the loopback address as a destination, the operating system returns the data back without sending traffic across any network

Private Addresses

Private IPv4 address ranges are reserved for private networks and can not go out to the public internet

Private addresses can be translated to one or more public addresses by using a technique called NAT and/or PAT

Address Resolution Protocol

ARP is a low-level protocol that hides the underlying network physical addressing, which allows an arbitrary IP address assignment to each host

  • Allows a host to find the physical address of a target host on the same physical network, given only the target IP address
  • A broadcast ARP request is used to find the hardware address of target hosts
  • To make ARP efficient, each host caches IP-to-physical address bindings

IP Routing

Routing is the mechanism that determines the path that data follows in order to travel across multiple networks from its source to its destination. Data is routed from its source to its destination through a series of routers, and across multiple networks

  • frames destined to the same physical network use direct delivery
  • frames destined to another physical network use indirect delivery
  • each host has a default router or gateway responsible for forwarding traffic to other networks

TCP/IP

Acronym that is commonly used for the set of network protocols that compose the Internet Protocol suite

libpcap

Portable and system independent API for capturing network traffic

  1. Written in C, but wrappers exist for many languages
  2. Supports saving captured packets to a file
  3. Supports reading files containing captured packets
  4. Supports a filtering mechanism based on the architecture of the BSD packet filter
  5. Applications include network statistics collection, security monitoring, network debugging, etc.

tcpdump

powerful command-line packet analyzer based on libpcap

  • places the NIC in promiscuous mode
  • may require root privileges, use sudo

// capture all packets on interface eth0 to screen
$ tcpdump -i eth0

// display captured packets in HEX and ASCII
$ tcpdump -XX -i eth0

// capture 10 packets up to 1500 bytes each on interface eth0, save them to file, filtering by host
$ tcpdump -c 10 -i eth0 -s 1500 -w data.pcap host www.academiadecodigo.org

// read packets from a previously saved file, be very verbose about it
$ tcpdump -vvv -r data.pcap
man tcpdump for additional options/features

Transport Layer

Provides end-to-end communication from one application program to another

  • may regulate flow of information
  • may provide reliable transport
  • must accept data from multiple user programs
  • programs within a host are addressed by port number

Ports

A port number is a way to identify a specific process to which an Internet or other network message is to be forwarded when it arrives at a server

Lower port numbers are reserved for standardized protocols, higher port numbers are free for use

User Datagram Protocol

UDP provides an unreliable connectionless delivery service using IP to transport messages between machines

It uses IP to carry messages, but adds the ability to distinguish among multiple destinations within a given host computer

Transmission Control Protocol

TCP provides a reliable full duplex connection between two machines, allowing them to exchange large volumes of data efficiently

  • Applications request a connection to the destination, which is used to transfer data;
  • Applications can handle multiple tcp connections simultaneously;
  • Guarantees that data will be delivered without loss, duplication or transmission errors;
  • Endpoints can exchange data in both directions simultaneously;
  • Guarantees reliable, synchronized startup between endpoints;
  • Guarantees delivery of all data after endpoint shutdown;
  • Provides flow control, allowing systems of varying speeds to communicate.

Client-server model

Architecture of a computer network in which many clients request a service from a centralized server (host computer). Servers wait for requests to arrive from clients and then respond to them.

Usually servers are more powerful machines located elsewhere on the network, but it is possible for the client and the server to reside on the same host

Sockets

Different processes can communicate with each other across networks by means of sockets. Java implements both TCP and UDP sockets, allowing the programmer to treat a network connection as just another stream.

java.net.DatagramSocket implements UDP sockets, capable of sending and receiving UDP datagrams to/from a specified host and port

java.net.Socket implements TCP client sockets, capable of connecting to a specified server and port

java.net.ServerSocket implements TCP server sockets, used to wait for requests to come in over the network.

sockets shield the programmer from low-level details, such as error detection, packet sizes, packet splitting, packet retransmission, network addresses and more

UDP Socket

Steps for sending/receiving to/from udp socket:

                        
// STEP1: Get your host and port
String hostName = args[0];
int portNumber = Integer.parseInt(args[1]);

// STEP2: Create send and receive buffers
byte[] sendBuffer = new byte[1024];
byte[] recvBuffer = new byte[1024];

// STEP3: Open a UDP (datagram) socket
DatagramSocket socket = new DatagramSocket(portNumber);

// STEP4.1: Create and receive UDP datagram packet from the socket
DatagramPacket receivePacket = new DatagramPacket(recvBuffer, recvBuffer.length);
socket.receive(receivePacket); // blocks while packet not received

// STEP4.2: Create and send UDP datagram packet from the socket
DatagramPacket sendPacket = new DatagramPacket(sendBuffer,
        sendBuffer.length, InetAddress.getByName(hostName), portNumber);
socket.send(sendPacket);

// STEP5: Close the socket
socket.close();
