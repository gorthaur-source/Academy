JavaBank

A Bank? Why?

A banking system covers a lot of the functionality and responsibilities of a typical enterprise application, such as:

  • User Interface
  • Client/Server Architecture
  • Relational Databases
  • Database Transactions

Domain Entities

The JavaBank application deals with three main entities:


Customer: Holds one
or more accounts

Recipient: receives money
transfers from other customers

Account: Holds a balance

Functional Requirements

The main features of the JavaBank application are:

  • Customer Management: Create, list, edit and delete customers
  • Account Management: Create, list, edit and delete accounts
  • Recipients Management: Create, list, edit and delete recipients
  • Money Transactions: Withdraw, deposit and transfer money between accounts or to recipients

Technical Requirements

JavaBank should be split into
a client and server

The user interface should be a web application - making it accessible through a web browser

Data should be stored in a persistent storage
