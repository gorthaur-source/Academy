    • Compiled vs Interpreted

    A compiled language is one where the program, once compiled, is expressed in the instructions of the target machine

    An interpreted language is one where the instructions are not directly executed by the target machine, but instead read and executed by some other program

    Compilers generate native machine code, interpreters read source code and generate machine code on the fly using some sort of run-time system

    Compiled Code

    • the main.c file contains the C source code to be compiled
    • compiler is a binary program that compiles C source code
    • main is the compiler output, an executable binary resulting from compiling the source code

    Interpreted Code

    • the index.js file contains the JavaScript source code to be interpreted
    • node.js is a binary program that interprets JavaScript source code

    Asymptotic Notation

    How to measure the speed of an algorithm?

    big O notation measures how fast a program run time grows as the size of the input increases towards infinity

    asymptotic notation does not depend on hardware performance

    linear search is O(n), as the time it takes to find element grows linear to the number of elements

    string length is O(n) if we need to count number of chars in string one by one

    string length is O(1) if length is stored before storing the string.

    binary search is O(log n) as for each time we double the number of input elements we need one more attempt to find the element
    big O is worst case y - time to run x - input size

    How about our algorithms?

    Linear Search
    O(n) time complexity

    Binary Search
    O(log n) time complexity

    Selection Sort
    O(n2) time complexity, inefficient in large lists

    Insertion Sort
    Worst Case - O(n2) time complexity
    Adaptive - O(n) time complexity when nearly sorted

    Binary Search - O(log n) because each time the input elements are doubled we need one more attempt to find the element.

    Data Structures

    Particular way of organizing and accessing data in a computer so that it can be used efficiently

    You already worked with a data structure, can you guess which? (Array)

    FILO and FIFO

    a Stack and a Queue

    Besides the array they are the most popular data structures. Stack: crumbs to be able to navigate backwards in state, browser history. CPU implements sub-routines (call/ret) to be able to return to the previous code execution address (instruction pointer)

    Tree

    Hierarchical data structure used to store, manipulate and search information that naturally forms a hierarchy

    Trees where each node can have a maximum of two children are called Binary Trees
    Example: The file system on a computer

    Graph Traversal Algorithm

    Find paths between a starting and a target node.

    Depth First Search visits child nodes first, requires usage of a Stack

    Breadth First Search visits neighbor nodes first, requires usage of a Queue

    Regular expressions define search patterns,

    think "I know, I'll use regular expressions."

    Regex Metacharacters

    Character Meaning Example
    * Match zero, one or more of the precious Ah* matches A or Ahhhhhhhhhhh
    ? Match zero or one of the previous Ah? matches Al or Ah
    + Match one or more of the previous Ah+ matches Ah or Ahhh but not A
    \ Used to escape a special character Hungry\? matches Hungry?
    . Wildcard, matches any character do.* matches dog, door, dot
    | Matches previous or next character/group (Mon)|(Tues)day matches Monday or Tuesday
    ( ) Groups characters See previous example
    [ ] Matches a range of characters [0-9]+ matches any positive integer
    { } Matches a specified number of occurrences [0-9]{2,4} matches 12, 1234 but not 12345
    ^ Beginning of a string or a negation if within a [] ^Hello matches string that begins with Hello
    [^0-9] matches any character not 0-9
    $ End of a string ing$ matches exciting but not ingenious
