Java Programming Environment

"Write Once, Run Anywhere"

Java is a Compiled/Interpreted language

Java is compiled into bytecode which is interpreted by the Java Virtual Machine (JVM).

JVM languages

Many computer programming languages are used to produce computer software that runs on the JVM

  • Clojure - a functional programming language
  • Groovy - a dynamic programming and scripting language
  • Scala - a statically-typed object-oriented and functional programming language
  • Kotlin - a statically-typed language from JetBrains, the developers of IntelliJ IDEA
  • JRuby - an implementation of Ruby
  • Jython - an implementation of Python

...and many more...

The Java Platform

  1. A platform is the environment in which a program runs;
  2. The Java platform has two components: The Java Virtual Machine; The Java Application Programming Interface (API)
  3. The API is a large collection of ready-made software components that provide many useful capabilities. It is grouped into libraries of related classes and interfaces; these libraries are known as packages.

The Java Memory Model

  • Each thread has its own stack
  • Local method variables reside on the stack
  • Objects reside on the heap
  • Just like a function, a Method is a collection of statements that are grouped together to perform an operation
  • An Object is a combination of variables and methods

Object-Oriented Language

  1. Java is an object-oriented language that take many of the features of C and C++ and improves upon them.

Programming Paradigms

  1. Procedural programming uses a list of instructions to tell the computer what to do step-by-step.
  2. Procedural programming relies on - you guessed it - procedures, also known as routines or subroutines.
  3. A procedure contains a series of computational steps to be carried out.
  4. Object-oriented programming, or OOP, is an approach to problem-solving where all computations are carried out using objects.
  5. An object is a component of a program that knows how to perform certain actions and how to interact with other elements of the program.
  6. Objects are the basic units of object-oriented programming.

Introduction to Java

Java is a statically typed language

variables and expressions have a type that is known at compile time

strongly typed: the variable type limits the values it can hold and the operations that we can perform on those variables

Variables

  • Primitive Types;
  • Reference Types.

Primitive Types

  • Built into the Java language and named by a reserved keyword
  • Do not have any functionality other than simple data storage

Primitive Types

Reference Types

  • Data types that reference objects
  • Reference data types are not built into the Java language like primitives
  • Can be provided as part of the Java SDK or written by the programmer
  • They store data and functionality associated with the data

Primitive Wrappers

Reference Types that add functionality to existing primitives


// Byte, Short, Integer, Long, Float, Double, Character, Boolean

// Primitive type
int primitiveInt = 29;

// wraps the primitive types inside objects
// new is used to create a new object, similar to {} in javascript
Integer objectInt = new Integer("29");

// the wrapper classes provide added functionality
System.out.println("To Binary: " + Integer.toBinaryString(primitiveInt));  // "11101"
System.out.println("To String: " + objectInt.toString());               // "29"
  1. the Java platform provides wrapper classes for each of the primitive data types;
  2. These classes "wrap" the primitive in an object;
  3. Autoboxing and unboxing. The automatic conversion between primitive and related class.

Operators

Assignment and Arithmetic Operators

  • Assignment operator is used to assign a value to a variable
  • Arithmetic operators are used to perform simple arithmetic operations on primitive data types

// assignment
int score = 200; // 200

// arithmetic operators
score = 100 + 5;	// 105
score = 100 - 5;	// 95
score = 100 * 5;	// 500
score = 100 / 5;	// 20
score = 100 % 3;	// 1 - modulus operator (%): (100 = 33 * 3 + 1)

Unary and Ternary Operators

  • Unary operators need only one operand
  • Ternary operators have three operands
  • Ternary operators are commonly used to shorthand if-else statements

int score = 100;

// pre and post increment/decrement
int highScore = score++; // highScore = 100 and score = 101
highScore = score--;  // highScore = 101 and score = 100
highScore = ++score; // highScore = 101 and score = 101
highScore = --score; // highScore = 100 and score = 100
// easy to insert Off-by-one type of bugs if not used on their own

score += 10; // 110 - a shorter version of score = score + 10
score -= 10; // 90
score *= 10; // 1000
score /= 10; // 10

// Usage of the ternary inside expression to replace if/else statement
int x = 10;
System.out.println((x % 2 == 0) ? "yes " + x : "no " + x);

x = 11;
System.out.println((x % 2 == 0) ? "yes " + x : "no " + x);

Relational Operators

  • Used to check for relations between values
  • Evaluate to a boolean value

score > 99 	// greater than
score < 56 	// less than
score >= 76	// greater than or equal to
score <= 83	// less than or equal to
score == 45	// equals
score != 23	// not equal to

Logical Operators

  • Used for testing multiple conditions
  • Evaluate to a boolean result

score > 99 && score < 150 // AND, returns true if all are true
score > 99 || score < 45 // OR,	returns true if at least one is true

Strings

A String consists of a sequence of characters

A String in Java is an Object, we access it using a string reference:

Creating Strings


// creating a String literal

// creating a String using the constructor

Which one should we use?

Go with String Literal


// using literals
String a = "Java";
String b = "Java";

System.out.println(a == b); // True, it is the same object, GO WITH THIS ONE

// using the constructor
String c = new String("Java");
String d = new String("Java");

System.out.println(c == d); // False, they are different objects

Compare the String content


// using literals
String a = "Java";
String b = "Java";

System.out.println(a.equals(b)); // True, they have the same content

// using the constructor
String c = new String("Java");
String d = new String("Java");

System.out.println(c.equals(d)); // True, they have the same content

Breaking from a Loop


String[] droids = { "M-TD", "C3PO", "R2-D2", "RA-7" };

// we are looking for R2-D2
for(int i = 0; i < droids.length; i++) {

    if(droids[i].equals("R2-D2")) {

        // do something with it and jump out of the loop
        System.out.println("This is the droid I was looking for...");

        break;

    }

    System.out.println("Still looking...");

}

The Continue


String[] droids = { "M-TD", "C3PO", "R2-D2", "RA-7" };

// looking for R2-D2 and C3PO
for(int i = 0; i < droids.length; i++) {

    if(!droids[i].equals("R2-D2") && !droids[i].equals("C3PO")) {

        // just skip the iteration
        System.out.println("This is not the droid I'm looking for...");

        continue;

    }

    // do something with the droid
    System.out.println("Found " + droids[i]);

}

Packages

Packaging Applications

The Java Archive (JAR) file format enables you to bundle multiple files into a single archive file containing the class files and auxiliary resources associated with the application.

To perform basic tasks with JAR files, you use the Java Archive Tool provided as part of the JDK

A JAR file is a zip archive containing a manifest file and potentially signature files

JAR files can be executed directly on OSX, Linux and Microsoft operating systems

Java Archive Metadata

The manifest is a special file that contains information about the files packaged in a JAR file. You can use the manifest for:

  • Setting the application entry point
  • Modifying the classpath
  • Setting the package version
  • Sealing the package
