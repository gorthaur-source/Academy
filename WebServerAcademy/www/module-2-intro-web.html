World Wide Web

System developed for information sharing across internationally dispersed teams, originally aimed at the High Energy Physics community

Stores documents and other web resources, interlinked with each other and accessed via the Internet

World Wide Web

In 1989, Tim Berners Lee invented the three fundamental technologies that are the foundation of the WWW

  • HTTP - HyperText Transfer Protocol
  • URI - Unique Resource Identifier
  • HTML - HyperText Markup Language

Not only that, he wrote the first web browser and first web server

After that, he created and still leads the World Wide Web Consortium (W3C)

Uniform Resource Locators

A pointer to a resource on the World Wide Web. Can be something as simple as a file or a directory, or a reference to a more complicated object, such as a query to a database or to a search engine

A Web HTML document can contain a number of URLs, pointing to other Web resources (links)

Hypertext Transfer Protocol

HTTP is a stateless, application-layer protocol for communicating between distributed systems and is the foundation of the modern web. Communication between a host and a client occurs via a request/response pair. The client initiates an HTTP request message, which is serviced through an HTTP response message in return.

Modern web apps use state, but they need to use other mechanisms to store state, as HTTP does not support it: (eg: cookie based sessions)

HTTP Verbs

HTTP verbs define the action to perform on a specific web resource

The most popular verbs are:

  • GET Fetch an existing resource. The URL contains all the necessary information the server needs to locate and return the resource
  • POST Create a new resource. POST requests usually carry a payload that specifies the data for the new resource
  • PUT Update an existing resource. The payload may contain the updated data for the resource
  • DELETE Delete an existing resource

HTTP Status Codes

With URLs and verbs, the client can initiate requests to the server. In return, the server responds with status codes and message payloads. The status code is important and tells the client how to interpret the server response

HTML

The most basic building block of the Web

Used to describe the content and structure of a web page

HTML 5 Elements

HTML documents consist of a series of elements used to enclose, wrap, or mark up different parts of the content to make it appear or act in a certain way

Anatomy of an HTML element

HTML Elements consist of

  • Opening Tag - The name of the element wrapped in opening and closing angle brackets. It states where the element begins or starts to take effect
  • Attributes - Modifies the default functionality of an element or provides functionality to certain elements unable to function correctly without them
  • Content - The content of the element
  • Closing Tag - States where the element ends or stops taking effect

The Document Tree

The Box Model

Each element in the HTML document is represented as a rectangular box, with the box's content, padding, border, and margin built up around one another

Block vs Inline elements

Two important categories of elements in HTML exist:

  • Block level elements form their own block. They use all available width and have their own rectangle on the browser window
  • Inline level elements do not have their own block. They live inside a containing block

Blocks are formatted vertically one after the other. Inlines are formatted horizontally

Allows the structure of a web page to be separated from its presentation and styling

Specifies how documents are styled, laid out and presented

Basic structure of a CSS rule

  • The selector specifies the html elements that will be affected by the rule
  • The declaration is a set of key/value pairs
  • Each key is an element property
  • Updating the element properties will update how the HTML content is displayed

Element Selectors

Select elements based on their type


/* Selects all p elements  */
p {
    border: 1px solid black;
}

/* Selects all h1 and h2 elements  */
h1, h2 {
    color: blue;
}

Class Selectors

The class attribute specifies one or more class names for an element


<p class="border">
    Lorem Ipsum
</p>

<h1 class="border header">
    HTML Styles
</h1>

Select elements based on their class


/* Selects all elements with class border */
.border {
    border: 1px solid black;
}

/* Selects all elements that contain both the border and header classes */
.border.header {
    color: blue;
    border: 1px solid blue;
}
more specific class selectors have precedence over less specific ones
it's because of the more specific precedence rule that elements with class border and header show up in blue and not black

ID Selectors

The id attribute specifies the id for an element, which needs to be unique in the document


<p id="border">
    Lorem Ipsum
</p>

Select elements based on their id


/* Selects the element with a border id */
#border {
    border: 1px solid black;
}
selectors have precedence over class selectors

Descendant Selectors

  • Selectors that utilize a descendant combinator to separate two other selectors are called descendant selectors
  • The descendant combinator is represented by a single space character
  • Elements matched by the second selector are selected if they have an ancestor element matching the first selector
 

<h1 class="title">
    <p>
        <ul>
            <li>Java</li>
            <li>JavaScript</li>
        </ul>
    </p>
</h1>

Add a red border around all list items that descend from elements of class title


.title li {
    border: 1px solid red;
}

Child Selectors

  • Selectors that utilize a child combinator to separate two other selectors are called child selectors
  • The child combinator is represented by a > character
  • Elements matched by the second selector are selected if they are direct descendants of elements matching the first selector
 

<h1 class="title">
    <p>
        <ul class="title">
            <li>Java<li>
        </ul>
    </p>
    <ul>
        <li>JavaScript<li>
    </ul>
</h1>
Add a red border around all list items that are direct descendants from elements of class title (Java only) 

.title > li {
    border: 1px solid red;
}

Inserting CSS into a web page

There are 3 ways of inserting CSS into a web page

  • External stylesheet
  • Internal stylesheet
  • Inline style

External stylesheet

The CSS rules are packed in its own file with the .css extension

The file should only contain CSS rules and no HTML

Changing a property on an external stylesheet will affect all webpages that import that stylesheet


<head>
    <link rel="stylesheet" type="text/css" href="mystyle.css">
</head>

Internal stylesheet

CSS rules included in the HTML file, enclosed by a <style> tag

The rules apply only to that specific HTML page


<head>
    <style>

        p {
            color: blue;
        }

        h1 {
            color: red;
            border: 1px;
        }

    </style>
</head>

Inline

The CSS rules are included as an attribute on an HTML element

The rules will apply only to that specific HTML element

Using inline css prevents the separation of style from structure and reutilization across elements and pages


<h1 style="color: red;">Hey I'm a red colored title</h1>

Rule priority

If you define different values to the same element's property the browser will determine its value based on this priority list:

  1. Inline style
  2. Internal stylesheet
  3. External stylesheet
  4. Browser defaults

Web Layouts

Besides floats, other techniques can be used to layout HTML elements on a page

  • tables are an ancient and outdated way of doing it, use modern css based alternatives instead
  • inline-block elements solve some of the problems with using floats, such as the need to clear them
  • flexbox is a modern css based layout technique, allowing the rapid creation of complex and flexible layouts
  • cssgrid is a modern css based layout technique that enables element alignment into columns and rows
  • tables should be used for tabular data and not layouts
  • Flexbox is essentially for laying out items in a single dimension – in a row OR a column
  • Grid is for layout of items in two dimensions – rows AND columns

Responsive Design

Comes from the necessity of designing websites that can be displayed on multiple devices with different screen sizes

Content must adapt and change its user interface depending on the browser or device accessing the site

    The techniques used to achieve a responsive web design are:

  • CSS3 Media Queries to use different styles based on screen size
  • Fluid Grid, a two-dimensional structure made up of a series of intersecting vertical and horizontal axes used to structure content
  • Flexible Images, which resize together with the design
Responsive designs can be tested with chrome dev tools

Grid System

Is a responsive mobile-first grid system that scales up to 12 columns as the viewport size increases

Content can span a different number of columns

Grid Classes

Screen sizes are divided into 4 types with different css prefixes

  • col-xs-* - extra small devices (phones, less than 768px, bootstrap default)
  • col-sm-* - small devices (tablets, 768px and up)
  • col-md-* - medium devices (desktops, 992px and up)
  • col-lg-* - large devices (large desktops, 1200px and up)

Bootstrap visibility

  • Bootstrap allows you to set the visibility of an element depending on screen size
  • An element can either be shown or hidden with the help of visible and hidden class prefixes
