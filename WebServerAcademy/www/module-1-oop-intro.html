Case Study

In the early 1970s, the US DoD commissioned a task force

Their mission was to investigate why the IT budget always went out of control without much to show for...

  • 80% of the budget went to the software, while the remaining 20% was spent on hardware
  • More than 80% of the software budget went to maintenance, only the remaining 20% was used on new software development
  • Hardware components could be shared and reused on other products
  • Software was often non-sharable and non reusable
  • Most often, the integrity of hardware did not affect other products, faults were isolated
  • Software faults could affect other running programs

The task force proposed to make software behave like hardware

An Object in Java

An object in Java is a container of properties and related behaviours that operate on those properties.

  • If you think about it, this is how we describe or think about almost every object we know.
  • To define an object we usually describe it’s properties: what it looks like, the several parts its has, etc;
  • And it’s behaviours: what the object is able to do or what we can do with it.
  • An object in Java is just a container of properties and related functionality.
  • But how do we create objects in Java? How do we know which properties or behaviours the object will have?

How do we create objects?

We need a set of specifications!

  • Well just like in real life, objects are created accordingly to a set of specifications.
  • If we take the calculator as an example, it is probably manufactured in a factory right?
  • But how does the factory know what it should do to build a calculator?
  • Well, the factory must have a set of designs and specifications that will guide them through the process, something like a template or a blueprint.

Classes and Instances

In Java, that set of specifications is called a Class and each object is an Instance of that Class.

each class is a new data type in the program

  • This design and specifications document it is what we call a class in Java.
  • So classes are the templates or blueprints from where objects are created.
  • With a single class we can create how many objects we need. Each object is also called an instance of the class.

Methods

A method is a function defined in a class. It consists of a group of statements that perform a specific task

In OOP, methods are the way to provide functionality to our objects

  • Imagine we want the calculator to perform some operations. We need for example a method to add numbers.
  • First we need to declare the method inside the calculator class and provide the necessary behaviour. Let’s see how it works.

Declaring a method


/**
* Calculator.java
*/

public class Calculator {

    public String brand;
    public String color;

    // declare the method add
    public void add() {

        int number1 = 1;

}
  • The Method Name allow us to identify and call the method.
  • It follows the same naming conventions of the variables however in this case we should always try to begin with a verb.
  • And the method body is where the actual behaviour happens.
  • In this case it declares two numbers, adds them and prints the result.
  • Variables declared within a method are called local variables.

Adding parameters to a method


/**
* Calculator.java
*/
public class Calculator {

    public String brand;
    public String color;

    // declare the method add with two parameters
    // method signature: name + parameters
    public void add(int number1, int number2) {

        // this makes no sense
        // int number1 = 1;
        // int number2 = 2;

        // print the result
        System.out.println(number1 + number2);
    }
}

We must always conform to the method signature!

  • What we need is a way to tell the method which numbers we want to add.
  • We can do this using parameters.
  • The Parameters allows us to pass variables to the methods body when calling it.
  • A parameter is a variable, so we need to declare it in the same way: with the data type and the name;
  • We can have as many parameters as we want, just need to separate them with a comma.
  • In this case the method is now accepting the two numbers it will add.
  • So we no longer need to declare the number variables inside the methods body.
  • Instead, the numbers being added are the ones passed in the parameters.
  • The method name plus the parameters is called the method signature.
  • Meaning, we can only pass parameters when they are defined and in such case we are obliged to do so.

Constructors

A constructor is a method called whenever we create a new object

It is responsible for initializing the object

  • In java constructors are instance methods with the same name as their class.
  • In java we create objects using the new operator.
  • After it creates our instance it will call its constructor

Default Constructor


/**
* Main.java
*/

public static void main(String[] args) {

    Calculator calculator = new Calculator(); // using the default constructor

    calculator.brand = "Casio";
    calculator.color = "Grey";

  • Objects come with a default (empty, no arguments) constructor. In fact we’ve already used it before.

  • THIS keyword in java that refers to the instance on which the method was called.

Encapsulation

Encapsulation refers to hiding data and implementation from the outside and providing methods to its access and use.

Why Encapsulation?

  • Provides an interface for the object behaviour;
  • Improves maintainability, flexibility and reusability;
  • Allows to implement read-only/write-only access.
  • We can add or remove the getter or setter to control the access control to the members;
  • We can perform validations on the data being inserted or choose how to return the data;
  • The user does not know how the data is stored; if we change the inner implementation, there is no need to change the code calling the getter.

Getters and Setters

  • Getter: returns the instance variable;
  • Setter: sets the new value to the instance variable.

Hiding the data


/**
* Calculator.java
*/

public class Calculator {

    // hiding the data from the "outside"
    private String brand;
    private String color;

    public Calculator(String brand, String color) {
        this.brand = brand;
        this.color = color;
    }

    // GETTER
    public String getBrand() {
        return this.brand;
    }

    // SETTER
    public void setBrand(String brand) {
        this.brand = brand;
    }
}
  • To hide the data we need to make it private. Making a variable being private means it is only accessible from the inside of the class.
  • If we try to access a private variable from the outside it will throw an error because the variable is no longer accessible.
  • We need to define methods to access the variables. These are usually called GETTERS and SETTERS.
  • A getter is a method that returns the value of a class variable.
  • While a setter is a method that assigns a new value to a class variable.

Accessing the data


/**
* Main.java
*/

public static void main(String[] args) {

    // using the new constructor
    Calculator calculator = new Calculator("Casio", "Grey");

    // using the SETTER
    calculator.setBrand("Texas Instruments");

    // using the GETTER
    String brand = calculator.getBrand();

    int result = calculator.add(3, 4);

    System.out.println(result);
}
  • So now to set the calculator brand we need to use the setter and to get the brand we use the getter.
  • Now you may be thinking, why are we going through all this trouble when we are doing exactly the same as before.
  • Another advantage of using encapsulation is that you can add additional verifications or behaviours when manipulating the variables.

Static Properties

We can create variables that are common to all instances of a class. They are associated with the class, rather than with a specific object.

We call them static or class variables and changing their value in one object will change its value for all objects

Instance vs Class variables


/**
* Calculator.java
*/

// Create a new class
public class Calculator {

    // class variables: common to all instances
    public static int calculatorsManufactured;

    // fields or instance variables: unique to each single instance
    private String brand; // Casio, Texas Instruments
    private String color; // Grey, Black
}

A public class variable is the equivalent of a Global variable

Global variables break the encapsulation principle and should be used carefully!
  • Each instance have their own copies of instance variables;
  • The values are stored in different memory locations;
  • Fields with the static modifier are associated with the class and not with the instances;
  • Class variables share one fixed location in memory;

Accessing Static Variables


/**
* Main.java
*/

public static void main(String[] args) {

    // create a Calculator instance
    Calculator calculator = new Calculator();

    // ...

    // class variables are referenced by the class name itself
    System.out.println(Calculator.calculatorsManufactured);
}
  • We access static variables using the class name.

Static Methods


/**
* Calculator.java
*/

public class Calculator {

    // class variables: common to all instances
    private static int calculatorsManufactured;

    // increment the number of calculators manufactured in the constructor
    public Calculator(String brand, String color) {
        this.brand = brand;
        this.color = color;
        calculatorsManufactured++;
    }

    // class method: obtain the total number of calculators manufactured
    public static int getCalculatorsManufactured() {

        return calculatorsManufactured;
    }
}

Can NOT access instance variables!

  • Static methods, which have the static modifier in their declarations, should be invoked with the class name, without the need for creating an instance of the class;
  • A common use for static methods is to access static fields
  • You can only access static variables inside static methods.

Using Static Methods


/**
* Main.java
*/

public static void main(String[] args) {

    // class methods are invoked with the class name
    System.out.println(Calculator.getCalculatorsManufactured());

}

Static methods are globally available

They do not require passing object references around, but...

They break the solid principles of OOP design and should be used carefully!
  • Not all combinations of instance and class variables and methods are allowed.

Instance and Static Methods

  • Instance methods can access instance variables and instance methods directly.
  • Instance methods can access class variables and class methods directly.
  • Class methods can access class variables and class methods directly.
  • Class methods cannot access instance variables or instance methods directly—they must use an object reference.
  • Also, class methods cannot use the this keyword as there is no instance for this to refer to.

Java Enumerations

An enum is a class that has a predefined set of instances 


/**
* DrinkType.java
*/

public enum DrinkType {
    BEER,
    VODKA,
    WATER
}
                    

// DrinkType is a java enumerated type
DrinkType currentDrink = DrinkType.BEER;

switch(currentDrink) {

    case BEER:
        System.out.println("Drinking beer.");
    break;

    case VODKA:
        System.out.println("Drinking vodka.");
    break;

}

Why Enumerations?

  • Type safety (enforced by the compiler)
  • Behaviour (just like ordinary classes)
  • Readability (DrinkType.BEER instead of 0)
  • Correctness (Input limited to predefined set of values)

When to use them?

Should always be used when a variable (especially a method parameter) can only take one out of a small set of possible values

  • The weapon type example is a good case for an enum. All values are known beforehand, meaning we know in advance the set of weapons that will exist in the game and that set of weapons won’t change.

Constructors and Instance Methods 


public enum Operation {
    ADD("+"), // Each enum value is actually a constructor call
    SUBTRACT("-"),
    MULTIPLY("x"),
    DIVIDE("/"); //end with ; when declaring class members afterwards

    private String symbol;

    Operation(String symbol) {
        this.symbol = symbol;
    }

    public String getSymbol() {
        return this.symbol;
    }
}
                    

public static void main(String[] args) {

    Operation add = Operation.ADD;
    System.out.println(add.getSymbol());
}

Enum Class Methods


public enum Operation {

    public static String showOperations() {
        String result = "Available Operations: ";

        // Operation.values() returns an array with all the enum values
        for(Operation operation : Operation.values()) {
            result += operation.symbol + " ";
        }

        return result;
    }
}
                    

/**
 * Main.java
*/

public static void main(String[] args) {

    System.out.println(Operation.showOperations()); // Available Operations: + - x /
